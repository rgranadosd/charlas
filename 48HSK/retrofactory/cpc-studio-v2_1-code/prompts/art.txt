REGLAS TÉCNICAS CRÍTICAS — MODE 0 SPRITE ENCODING:
- 1 byte = 2 píxeles consecutivos con bits INTERCALADOS. El valor del pen NO es el valor del byte.
- Usar la tabla de conversión pen→byte para cada par de píxeles.
- NUNCA usar pen 0 como fill interior: pen 0 = fondo (SKY_BLUE, invisible contra el cielo).
- Fill opaco: pen 1 (BLACK) = byte 0xC0 cuando ambos píxeles son pen 1.
- Contorno: pen 6 (WHITE) = byte 0x3C cuando ambos píxeles son pen 6.
- Contorno brillante: pen 4 (BRIGHT_WHITE) = byte 0x30.

Tabla de valores más usados (un solo pen para ambos píxeles):
  pen0=0x00 (fondo, INVISIBLE)  pen1=0xC0 (BLACK, fill opaco)
  pen2=0x0C (ORANGE)            pen3=0xCC (BRIGHT_RED)
  pen4=0x30 (BRIGHT_WHITE)      pen5=0xF0 (BRIGHT_YELLOW)
  pen6=0x3C (WHITE)             pen7=0xFC (CYAN)

⚠ pen1+pen1 = 0xC0 (BLACK), NO 0x03 (eso es pen8+pen8 = BRIGHT_BLUE)
